# 技术报告：基于文心大模型的移动端个人 AI 助手

## 1. 系统架构

本系统采用「云端大脑 + 端侧执行」的端云协同架构：

- 云端：百度文心大模型（ERNIE-4.5-Turbo-32K）作为核心推理引擎，通过星河社区 API 提供自然语言理解与生成能力
- 端侧：天玑 9500 Android 设备运行 OpenClaw Gateway（基于 Termux + Node.js），负责会话管理、工具调度、文件操作等本地任务
- 通信：WebSocket 长连接（ws://127.0.0.1:18789）确保端云实时交互

## 2. 关键技术

### 2.1 文心大模型 API 接入
- 使用 OpenAI 兼容接口，通过 baseUrl 指向星河社区端点
- 模型选型：ernie-4.5-turbo-32k（128K 上下文，2048 max_tokens）
- 解决 API 参数限制（max_completion_tokens ≤ 2048）和权限问题

### 2.2 OpenClaw 移动端适配
- 在 Termux 中部署 Node.js 环境，解决原生模块编译问题
- 通过配置文件（openclaw.json）管理模型、端口、认证信息
- 后台运行 Gateway（nohup），保障服务稳定性

### 2.3 任务调度与工具调用
- Agent 通过自然语言指令执行任务，底层调用 OpenClaw 工具链
- 支持文件操作、系统命令执行、提醒设置等功能
- 通过 /new、/compact 等命令管理会话上下文，防止溢出

## 3. 创新点

### 3.1 轻量级端云协同架构
- 利用手机 Termux 环境运行完整 Gateway，无需 root 或特殊权限
- 云端模型处理自然语言，端侧执行具体操作，降低延迟

### 3.2 可复现的部署方案
- 从零开始的环境搭建指南，覆盖 Windows 开发验证与 Android 真机部署
- 配置文件脱敏处理，保护 API 密钥安全

## 4. 多模态融合意图识别（预留说明）

本项目当前版本聚焦文本对话与任务执行，尚未集成多模态能力。后续可通过以下方式扩展：
- 在天玑 9500 上利用 NPU 运行轻量级视觉/语音模型，实现端侧感知
- 将图像、语音特征作为上下文输入文心大模型，实现多模态意图理解

## 5. 智能体动态编排与任务调度机制

- OpenClaw 的 Skills 系统允许动态加载技能插件，扩展 Agent 能力
- Heartbeat 机制支持主动行为调度，实现定时提醒等自动化任务
- Gateway 支持多 Agent 管理，可编排不同 Agent 协同工作

## 6. 性能指标

- 内存占用：< 2GB
- 连续运行：> 30 分钟无崩溃
- API 延迟：30~120 秒（取决于任务复杂度）
- 任务成功率：基础对话 100%，工具调用成功率较高
